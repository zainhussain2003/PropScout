C · Proposals (built faithful; your call)

Each was built the faithful way; these are places the current IA is arguably wrong.

Proposal 01  Investor §02 gap. The live renderer numbers 01, 03…08 — §02 (Financing sliders) exists only on the standalone demo page, yet the hero still reads "Live recalc · sliders below." Either wire FinancingSliders in as §02 or renumber. Faithful build preserves the gap.

Proposal 02  Duplicate n in InvestmentMetricsSection. The shared component hardcodes n="01"; LandlordPage.tsx reuses it as its §02 → the live landlord demo prints "§ 01" twice, and STR + checklist both print "§ 11". Fixed on the design surface by adding an n prop (default "01"); propose the same upstream.

Proposal 03  Landlord renderer. ReportPage.tsx routes landlord mode through the investor renderer, which degrades on a for-rent listing (no sale price → "metrics unavailable"). The design follows LandlordPage.tsx (the designed §01–§12 surface) instead — the largest interpretation in the set.

Proposal 04  Tenant report scope. Live TenantReportContent is a 4-section summary; the tenant demo page has extra negotiation/cost/leverage sections. Faithful build keeps the 4-section live version; target-offer figures ($2,450–2,500, $1,800–2,400 saved) live in the AI verdict copy.
